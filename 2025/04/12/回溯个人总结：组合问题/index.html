回溯个人总结：组合问题

算是个人刷代码随想录的个人总结吧

理论基础

回溯三部曲：

  1. 确定回溯函数模版以及返回参数
void backtracking(参数)
  1. 确定回溯函数终止条件
if (终止条件) {
	存放结果;
	return;
}
  1. 回溯搜索的遍历过程
for (选择：本层集合中元素（树中节点孩子的数量就是集合的大小）) {
    处理节点;
    backtracking(路径，选择列表); // 递归
    回溯，撤销处理结果
}

回溯模版：

void backtracking(参数) {
    if (终止条件) {
        存放结果;
        return;
    }

    for (选择：本层集合中元素（树中节点孩子的数量就是集合的大小）) {
        处理节点;
        backtracking(路径，选择列表); // 递归
        回溯，撤销处理结果
    }
}

77. 组合

给定两个整数 n 和 k，返回范围 [1, n] 中所有可能的 k 个数的组合。
你可以按 任何顺序 返回答案。

示例1：
输入： n = 4, k = 2
输出： [[2,4],[3,4],[2,3],[1,2],[1,3],[1,4]]

示例2：
输入： n = 1, k = 1
输出： [[1]]

提示：

  • 1 <= n <= 20
  • 1 <= k <= n

问题分析

按照示例1的测试用例，画出其层级结构如下：

1、确定回溯函数模版以及返回参数

全局变量：

  • 结果集：List<List<Integer>> res = new ArrayList<>();
  • 保存每一次遍历后得到的结果：List<Integer> list = new ArrayList<>();

函数参数：

因为我们每一次遍历都需要控制每一棵树（或子树）的root（如上图，三棵树的root分别为：1、2、3），所以传入回溯函数的参数里，除了n和k，还有index：

public void backTracking(int n, int k, int index);

2、确定回溯函数终止条件

终止条件：

  • 当list的大小正好等于k时，把结果添加到结果集，返回
if (list.size() == k) {
    res.add(new ArrayList<>(list));
    return;
}
  • 注意，这里需要新建一个ArrayList，因为list是全局变量，在递归中会重复使用，如果不新建一个ArrayList，后续对list的改变会影响到已经加入到结果集里的结果

3、分析回溯搜索的遍历过程

再贴上此图：

  • 一层一层看，每一层的节点个数就是for循环里要遍历的次数
  • 从上往下看，树结构的层数，就是递归遍历的次数

组合里的元素不可重复取（[1, 2] 和 [2, 1] 是同个组合），所以我们都只需要往后取元素，不用往前看。

  • index = 1 时，就不需要取 index - 1 的元素了，因为 index = 0 时已经取过 index + 1 的元素了

综上，构建出回溯循环：

for (int i=index; i<=n; i++) {
    // 处理节点
    list.add(i);
    // 递归
    backTracking(n, k, i+1);
    // 回溯
    list.remove(list.size()-1);
}

初始答案

附上：

class Solution {
    public List<List<Integer>> res = new ArrayList<>();
    public List<Integer> list = new ArrayList<>();

    public List<List<Integer>> combine(int n, int k) {
        backTracking(n, k, 1);
        return res;
    }

    public void backTracking(int n, int k, int index) {
        // 终止条件
        if (list.size() == k) {
            res.add(new ArrayList<>(list));
            return;
        }

        // 循环
        for (int i=index; i<=n; i++) {
            list.add(i);
            // 回溯
            backTracking(n, k, i+1);
            list.remove(list.size()-1);
        }
    }
}

剪枝优化

当我们剩余可以取的元素已经小于我们需要取的元素时，就可以跳出循环了
优化过程：

  1. 已经选择的元素个数：list.size();
  2. 还需要的元素个数为: k - list.size();
  3. 在集合n中至多要从该起始位置 : n - (k - list.size()) + 1，开始遍历
class Solution {
    public List<List<Integer>> res = new ArrayList<>();
    public List<Integer> list = new ArrayList<>();

    public List<List<Integer>> combine(int n, int k) {
        backTracking(n, k, 1);
        return res;
    }

    public void backTracking(int n, int k, int index) {
        // 终止条件
        if (list.size() == k) {
            res.add(new ArrayList<>(list));
            return;
        }

        // 循环
        for (int i=index; i<=n-(k-list.size())+1; i++) {
            list.add(i);
            // 回溯
            backTracking(n, k, i+1);
            list.remove(list.size()-1);
        }
    }
}

执行用时优化：21ms -> 19ms

39. 组合总和

给你一个 无重复元素 的整数数组 candidates 和一个目标整数 target ，找出 candidates 中可以使数字和为目标数 target 的 所有 不同组合 ，并以列表形式返回。你可以按 任意顺序 返回这些组合。
candidates 中的 同一个 数字可以 无限制重复被选取 。如果至少一个数字的被选数量不同，则两种组合是不同的。
对于给定的输入，保证和为 target 的不同组合数少于 150 个。

示例1：
输入： candidates = [2,3,6,7], target = 7
输出： [[2,2,3],[7]]
解释：
2 和 3 可以形成一组候选，2 + 2 + 3 = 7 。注意 2 可以使用多次。
7 也是一个候选， 7 = 7 。
仅有这两种组合。

示例2：
输入: candidates = [2,3,5], target = 8
输出: [[2,2,2,2],[2,3,3],[3,5]]

示例3：
输入: candidates = [2], target = 1
输出: []

提示：

  • 1 <= candidates.length <= 30
  • 2 <= candidates[i] <= 40
  • candidates 的所有元素 互不相同
  • 1 <= target <= 40

问题分析

这题和上一题不太一样的是，这题里的元素可以无限制重复被选取

还是一样，根据测试用例1先画出层级结构：

这层级结构一看，后面肯定是需要剪枝的。。但是我们先用纯暴力，把整体思路写出来

1、确定回溯函数模版以及返回参数

全局变量：

  • 结果集：List<List<Integer>> res = new ArrayList<>();
  • 保存每一次遍历后得到的结果：List<Integer> list = new ArrayList<>();

回溯函数参数：

和上一题一样，除了包括题目的输入里的两个参数，还有每棵树的root，我们用index控制；此外，我们要计算遍历到此的所有节点的总和，即 sum

  • int[] candidates
  • int target
  • int index
  • int sum
public void backTracking(int n, int k, int index, int sum);

2、确定回溯函数终止条件

  1. 当 sum > target 时，直接返回（当做剪枝了）
  2. 当 sum == target 时，添加到结果集再返回
// 终止条件
if (sum > target) {
    return;
}
if (sum == target) {
    res.add(new ArrayList<>(list));
    return;
}

3、分析回溯搜索的遍历过程

前面提到，这题和上一题不太一样的是，这题里的元素可以无限制重复被选取

如上图，树的每一层元素都是从第一个元素开始添加的，而这个是由for循环里的 i 来控制的，所以：

for (int i=index; i<candidates.length; i++) {
    list.add(candidates[i]);
    backTracking(candidates, target, i, sum + candidates[i]);   // 可以重复选取，不用i+1
    list.remove(list.size()-1);
}

答案

附上：

class Solution {
    List<List<Integer>> res = new ArrayList<>();
    List<Integer> list = new ArrayList<>();

    public List<List<Integer>> combinationSum(int[] candidates, int target) {
        backTracking(candidates, target, 0, 0);
        return res;
    }

    public void backTracking(int[] candidates, int target, int index, int sum) {
        // 终止条件
        if (sum > target) {
            return;
        }
        if (sum == target) {
            res.add(new ArrayList<>(list));
            return;
        }

        // 循环
        for (int i=index; i<candidates.length; i++) {
            list.add(candidates[i]);
            backTracking(candidates, target, i, sum + candidates[i]);   // 可以重复选取，不用i+1
            list.remove(list.size()-1);
        }
    }
}

40. 组合总和II

给定一个候选人编号的集合 candidates 和一个目标数 target ，找出 candidates 中所有可以使数字和为 target 的组合。
candidates 中的每个数字在每个组合中只能使用 一次 。

注意： 解集不能包含重复的组合。

示例1：
输入: candidates = [10,1,2,7,6,1,5], target = 8
输出: [[1,1,6],[1,2,5],[1,7],[2,6]]

示例2：
输入: candidates = [2,5,2,1,2], target = 5
输出: [[1,2,2],[5]]

提示:

  • 1 <= candidates.length <= 100
  • 1 <= candidates[i] <= 50
  • 1 <= target <= 30

问题分析

和上题又有点不同的是：

  1. 每个元素只能用一次
  2. 结果集要去重

我们可以先把 candidates 排序，方便后续去重

从层级结构可以看出，当我们遍历到当前层已经遍历过了的元素，直接continue即可

如下：

class Solution {
    List<List<Integer>> res = new ArrayList<>();
    List<Integer> list = new ArrayList<>();

    public List<List<Integer>> combinationSum2(int[] candidates, int target) {
        // 排序
        Arrays.sort(candidates);
        backTracking(candidates, target, 0, 0);
        return res;
    }

    public void backTracking(int[] candidates, int target, int index, int sum) {
        // 终止条件
        if (sum > target) {
            return;
        }
        if (sum == target) {
            res.add(new ArrayList<>(list));
            return;
        }

        // 循环
        for (int i=index; i<candidates.length; i++) {
            // 对同一层已经使用过的元素跳过
            if (i>index && candidates[i-1]==candidates[i]) {
                continue;
            }
            list.add(candidates[i]);
            backTracking(candidates, target, i+1, sum+candidates[i]);
            list.remove(list.size()-1);
        }
    }
}

尝试总结

组合问题里，即返回的结果集里没有顺序，[1, 2]和[2, 1]是一个组合，这组题目里还需要关注几种情况：

  1. 可以重复选吗？
    • 不可重复选：backTracking(i + 1)
    • 需要重复选：backTracking(i)
  2. 每次处理完节点后还需要做什么别的操作吗？（比如计算总和）
    • 需要计算总和，需要多传入参数：backTracking(i, sum + temp)
  3. 会不会造成结果集重复的问题？
    • 要根据具体题目进行分析、去重
